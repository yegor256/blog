    A Remote Slave Is Still a Slave

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    远程办公无疑是一种趋势，根据劳工统计局（BLS）和我的个人观察。让员工在家工作似乎是许多初创企业和一些大公司，如Buffer、Automattic、Groove和其他许多公司的万能解决方案。然而，在大多数情况下，用虚拟办公室取代传统的实体办公室并不能帮助公司和雇员变得更加高效。

    在家工作，也被称为随地办公、远程工作或简单地称为远程办公，通常会导致四个重大变化：

    • 其次，我们用Zoom或Skype等会议软件取代了会议室，并学习如何使虚拟会议更加高效。其主要原则与传统会议相似：让它变得有趣。否则，当经理仍在她的窗口中发言时，我们将转向Facebook或PokerStars。

    • 第三，我们用Trello或其他任何 任务跟踪器来替换便利贴和Scrum板。我们让每个人都忙于学习新工具，并用数据填充它们。

    • 第四，也是最重要的是，我们用Slack频道或HipChat群组取代了办公室闲聊。每个人不需要在早上9点到办公室，而是可以通过表情符号展示他们的出勤率水平。

    作为一名开发者，在一个远程工作的团队中，你的生活将会非常不同。它将充满真正的自由。然而，也会有一些问题：

    很难证明自己的清白。你的经理如何知道你真的在工作，而不是在看电视、和孩子玩，或者在照顾你最喜欢的开源宠物？在办公室里，这更容易；你只需要坐在显示器前，老板就满意了。但是在家或咖啡店里，你默认就是有罪的，直到你通过创建新的工单、在Slack上发布表情符号和发送带有多个抄送的电子邮件来证明相反。

    很难找到信息。在办公室里，你总是可以大声提问，让最有知识的人来帮助你。然而，在远程工作中，你将不得不搜索维基页面、Google文档，甚至是Git日志。要准备好几乎没有人愿意分享知识或远程教导你——如果没有人看到他这样做，那有什么意义呢？当你面对那些一年前由某人在写完后就被解雇的文件时，你将孤立无援。

    很难避免加班。在办公室里，当时间到了，你只需和其他人一起离开。而在远程工作中，你将在周日深夜接到电话和Slack讨论，特别是如果团队分布在多个时区。每个人都期望你24小时全天候都可用，因为你在家工作。而在Skype中将状态改为“请勿打扰”只会冒犯他们，严重损害你的声誉。在远程工作环境中，优秀的程序员是那些总是在线的人。

    很难为自己辩护。你不会知道下一次麻烦会从哪里来。在办公室里，你可以通过目视观察领域、嗅到空气中的冲突、看到人们的行动、他们谈论的内容以及谁是下一个受害者来发现。而在远程工作中，你独自坐在孤立中，从Slack中提取信息。你甚至可能在毫不察觉的情况下成为下一个受害者。简而言之，当你写Java并思考新的单元测试时，他们会在你背后捅你一刀。

    很难不浪费时间。听起来有些违反直觉，但是在家里坐着，你会比在办公室浪费更多的时间。会议将更频繁举行，由于日程安排问题、连接质量问题、软件故障以及缺乏在线沟通技巧，它们将变得不那么有效。经理将更多地召开会议，主要是为了说服自己团队实际在工作；每天多次听到你的声音，是你忙碌和他是一位好经理的完美证明。

    在更严肃的问题上，我认为当团队的动力仍然基于按小时支付时，远程工作只会使事情变得更糟。远程工作的奴隶仍然是奴隶。而且，这个奴隶更难管理和控制。首先，我们必须转向基于结果的支付模式，然后再实现远程工作。实际上，只要每个人开始拿到他们交付的成果的薪水，而不是他们花费的时间，远程工作就会自动发生。他们将停止出现在办公室，但他们的成果将继续为项目做出贡献，而无需任何网络摄像头或Slack频道。

    不足为奇，许多大公司已经决定停止这种远程办公的荒谬行为，将他们的奴隶重新关在闭门之后；以Google、Yahoo和Best Buy为例。
