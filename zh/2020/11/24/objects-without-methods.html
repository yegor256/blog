    Objects Without Methods

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    你认为面向对象编程中的对象是什么？无论你使用什么编程语言，你很可能会同意Bruce Eckel在《Java编程思想》中所说的，”每个对象都有一个状态和可以要求它执行的操作”，或者Benjamin Evans在《Java核心技术》中所称的，它是”一组保存值并对这些值进行操作的数据字段的集合”。然而，请稍等…如果我告诉你一个对象可能没有”操作”，但仍然可以是David West在他的著作《面向对象思考》中所建议的”宇宙构建的等效物”，你会怎么想？

    在EO中，我们的实验性编程语言，我们试图重新定义面向对象编程（OOP）及其对象。EO中有两种类型的things：原子和对象。原子是最低级的语言原语，无法由其他原子表示。例如，两个对象的算术加法是一个原子（跟着我，这是EO语法，你会习惯的）：

    add 5 y > x

    In a more traditional Java-like infix notation this code would look like this:

    x = add(5, y)

    原子是add，它的两个特定参数是5和y。该语句使用现有的原子并指定其参数来创建一个新的原子。正在创建的新原子的名称是x。一旦我们要求这个新创建的原子做任何事情，它会获取y中的内容，加上5，然后开始像它们的总结一样表现。在那之前，它保持安静。EO是一种声明性语言。

    原子由 EO 运行时提供。例如，add、sub、mul 和 div 用于算术运算；if 和 for 用于分叉和迭代；less、and、eq、or 用于逻辑操作，等等。原子可能类似于带有参数的低级函数。然而，它们不会立即计算结果，而是在需要时才计算。说add(5, file) 不会立即导致读取文件内容并将其加上 5。只有在处理创建的原子时才会发生文件读取。

    Next, on top of these atoms, objects can be created by a programmer. For example, this is an object that represents a circle:

    [r] > circle
  mul 2 3.14 r > perimeter
  mul 3.14 r r > area

    第一行创建了一个名为circle的”抽象”对象。它是抽象的，因为它的一个属性r是”自由”的。在这个对象中没有指定r，所以不能直接使用这个对象，必须使用指定了r的副本。例如，这是半径为30的圆c。

    circle 30 > c

    对象circle有三个属性。第一个是r，它是自由的。另外两个是perimeter和area。它们是”bounded”的，因为它们的原子已经定义好了：在两种情况下都是mul。要获得圆c的面积，我们这样做：

    c.area > a

    看起来像是一个方法调用，但实际上并不是。我们不会调用一个方法，我们只是从对象 c 中取出一个 area 对象。当我们执行 c.area 时，并不是在那一刻为我们创建了它！它已经存在并等待我们取走它。它是在对象 c 构建时就已经创建好了。

    这是Java中的方法与EO中的属性之间的区别。在Java中，每个方法都是一种在调用时立即执行的过程。这种方法调用（或消息发送，根据早期面向对象编程的爱好者）机制是从C函数继承而来，而C函数本身则是从ALGOL过程继承而来，我相信如此。EO处理方式不同。它没有方法调用。它只是从对象中取出属性并将其传递给其他对象，直到控制权传递给它们并降至原子级别。

    在上面的例子中，对象a不是一个计算出的数字。它是一个封装了3.14和30（半径）的原子mul。计算的结果尚未知道。如果我们不对a做任何操作，CPU将永远不会进行计算。然而，如果我们决定将数字打印到控制台，计算将会发生。

    stdout
  sprintf
    "Radius is %d, Area is %d"
    r
    a

    在这里，原子sprintf构建了字符串，该字符串封装了三个属性：文本、r和a。顺便提一下，可以使用垂直或水平符号来构建对象。上面的代码可以这样写：

    stdout (sprintf "Radius is %d, Area is %d" r a)

    atom stdout 封装了由 sprintf 构建的字符串，并保持静默。它不会打印任何东西！只有当某人在某个时刻尝试从该对象中取出其属性时，atom stdout 才会将该行内容输出到控制台上。

    在stdout、sprintf、mul和大多数其他原子中都没有属性，除了一个：𝜑。任何对象或原子都有这个特定的属性，也被称为对象的“主体”。一旦有人尝试触摸stdout.𝜑，控制台将看到字符串。

    Thus, we have objects, but we don’t have methods. There are only attributes representing other objects.
