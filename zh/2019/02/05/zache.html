    Zache: A Simple Ruby In-Memory Cache

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    一个月前我遇到了一个问题：我找不到一个能够在内存中缓存并能够在超时时过期的Ruby gem。经过一些快速调查，我决定自己实现一个，并称其为Zache（意为“零缓存”，因为没有后端）。以下是它的工作原理：

    首先，您创建缓存：

    require 'zache'
zache = Zache.new

    然后，您通过键获取值，同时提供将在键不存在或过期时执行的代码块：

    x = zache.get(:x, lifetime: 15) do
  # Something very slow and expensive, which
  # we only want to execute once every 15 seconds.
end

    在这里，:x 是键，15 是它在缓存中保留的秒数，直到过期。

    请注意，键不会自动从缓存中删除。它会一直保留在那里，直到下一次调用get(:x)为止。只有在那个时刻，它才会被标记为”过期”。为了清理缓存，您可以调用zache.clean()，所有过期的键将被删除。

    你可以在一个独立的线程中定期执行它，例如，每分钟执行一次。

    Thread.start do
  sleep 60
  zache.clean
end

    当然，Zache是线程安全的。
