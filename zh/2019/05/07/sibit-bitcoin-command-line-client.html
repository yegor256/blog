    Sibit Demonstrates How Bitcoin Works

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    比特币 对我来说是一个巨大的技术谜团。我读过的所有关于比特币的文章听起来都非常复杂，难以理解。直到我被分派了一个任务：我必须将Zold，我们的实验性非区块链加密货币，与比特币整合在一起。我不得不研究比特币的架构，然后我找到了这个简短而简单的视频（我强烈推荐你观看）。我成功地实现了整合，并理解了区块链的工作原理。以下是我的简短总结，希望对你有所帮助。

    首先，有一个私钥，它是一段简短的文本，例如：

    c93a36feb31712c390a78b37337cb85d45d3b2f9f6e55108bde32477cbabac5f

    我是如何生成的？ 它是随机的。 你也可以生成一个。 安装Sibit，我的Ruby宝石，并运行以下命令（它是一个命令行工具）：

    $ sibit generate
c93a36feb31712c390a78b37337cb85d45d3b2f9f6e55108bde32477cbabac5f

    每次调用sibit generate，您都会获得一个新的私钥。它只是一段随机的文本（实际上是一个大的十六进制数）。然后，您可以从私钥创建一个地址。每个私钥都有且只有一个地址。以下是创建地址的步骤：

    $ sibit create c93a36feb31712c390a78b37337cb85d45d3b2f9f6e55108bde32477cbabac5f
1K3JgsdRbbUDUgTGo4gvUXUzRZjBse1TYe

    你把它交给想要向你付款的人。比特币交易并不是你想象中的银行交易。比特币交易具有若干“输入”和“输出”地址。比如，三个输入地址每个都是10个比特币，而两个输出地址分别是25个比特币和5个比特币。所有输入的总和必须等于所有输出的总和。

    因此，在一次交易中，您可以在地址之间转移大量数字货币。您可以将一百个输入地址发送到另一百个输出地址。当我发现这一点时，这让我有点惊讶。交易的大小可能相当大。大小很重要。它们以字节计算。一笔只有一个输入和一个输出的交易将消耗大约220字节。请记住这个数字，我们很快会回到它。

    正如你能想象的那样，为了计算你在某个地址上拥有多少钱，你只需遍历整个数据库中的所有交易（目前已有超过4亿次交易），并查看有多少笔交易的输出是你的地址（输入）。 然后你扣除任何你的输出和别人的输入的交易，得到的就是余额。有很多公共和免费的网络服务可以为你完成这项工作。你不需要自己搜索整个数据库，只需使用Blockchain API（例如），他们将为你提供有关任何比特币地址的信息。例如，试试这个地址：

    $ sibit balance 1MUhYhaBqzgpwL1focqJNhTymdXbyky9UY
20000000

    这是以satoshi为单位的，相当于0.2 BTC。点击这里，您将看到此地址的输入和输出。只有一笔交易提到了这个地址。该交易有一个输入和两个输出。其中一个输出是我的。我得到了0.2比特币，它们仍然在这里，因为没有将它们转移到其他地方的交易。

    因此，简单来说，区块链数据库是一个包含输入和输出的交易大列表。一旦有新的交易进入该数据库，几个地址的余额就会发生变化。该数据库只会增长，之前的交易无法被删除或修改。因此，一旦您的交易进入，资金就会从一个地址转移到另一个地址。

    要创建一个交易，您需要知道所有输入地址、它们的私钥以及所有输出地址。如果您希望发送的金额大于您在单个地址中拥有的金额，您需要多个输入。比特币中有一个称为钱包的东西，它是一种软件，比如Electrum，它可以跟踪您的所有地址，并知道如何创建一个交易，将多个地址组合起来并将它们用作输入。就是这样。

    所以，你知道输入，你指定输出，并且预计保留一小部分未花费的金额。假设你的输入有10,000聪，你为8,000指定一个输出。你保留了剩下的2,000未花费。它们被称为矿工费。你将交易发送到所有比特币节点，它们将其放入内存池，并尝试将几千个交易组合在一起形成区块。它们一起尝试，只有其中一个成功，大约每10分钟一次。一旦获胜者成功解决一个区块，它就会赢得你交易中未触及的那2,000。区块中的每个交易都支付给获胜者一些小费用。

    有可能一分钱不用付吗？是的。然而，您在交易中留下的手续费越小，您的交易进入下一个区块的机会就越低。每个矿工都希望从每个区块中获得最大利益，并在尝试挖矿时将最“慷慨”的交易放入区块中。

    费用取决于交易的字节数。由于任何交易可能包含多个输入和输出，只有其大小才是重要的。例如，我在此交易中支付的费用为8,971 satoshi，大约为每字节40 satoshi，因为我的交易包括225字节。

    将您的交易发送给所有比特币节点有两种选择。第一种选择是启动您自己的全节点，它将通过比特币协议与其他节点进行通信。第二种选择是使用其中一个HTTP中继，您只需在POST HTTP请求中发布您的交易，中继将把它发送给所有比特币节点；这是其中之一。

    Sibit自动化了这个过程。您只需说：

    $ sibit pay 200000 L \
  1K3J...se1TYe:c93a36feb3...e55108bde32477cbabac5f \
  153dF1xKyVX5X8brBDroaqKkPmcB8kLtDB \
  1K3JgsdRbbUDUgTGo4gvUXUzRZjBse1TYe

    这里，第一个参数是您要发送的金额。第二个参数是您希望保持不变的费用。我为您简化了操作，以帮助您避免计算。只需说S、M、L或XL，费用将被计算得更或多或少正确。第三个参数是地址和私钥的列表，用冒号分隔。第四个参数是目标地址（Sibit使得仅能发送到一个地址）。最后一个参数是将发送找零的地址。

    如果您的输入中有50,000 satoshi并且您想要发送15,000 satoshi给朋友，您必须找到一个地方将其他35,000 satoshi发送过去——当然，扣除矿工费用。您只需提供您自己的地址，找零将会发送到那里。

    这就是你需要了解有关比特币和区块链的全部内容，以便使用它。我漏掉了什么？
